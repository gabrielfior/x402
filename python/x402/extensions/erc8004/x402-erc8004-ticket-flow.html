x402 × ERC-8004 — ticket flow

1-step x402 (same URL). TicketMinter + ReputationRegistry v3. Facilitator sponsors settlement/finalize gas.

Client — payer, request/response digests, feedback tx Agent server — 402, handler, optional receipt sig Facilitator — verify + on-chain settle/mint/finalize (gas) On-chain — TicketMinter, IdentityRegistry, ReputationRegistry
Ticket: NONE → MINTED (paid + request) → FINALIZED (response) → CONSUMED (feedback)

Phases (one paid HTTP request)

0 — Discover x402

Client GET /resource → 402 with extensions.erc8004.agentId, accepts (asset, amount, payTo). Client learns endpoint (this URL) only from the server.

1 — Bind request (off-chain, before retry)

Client computes requestHash, interactionHash (includes settlement + request digests). Signs payment auth (EIP-3009 / Permit2 / ERC-20).

2 — Paid retry x402

Same GET /resource + X-PAYMENT. Server middleware: verify → settle (via facilitator).

3 — Settle + mint on-chain

Facilitator calls TicketMinter.settleAndMintTicket(...) (one tx, facilitator gas). Atomic: ERC-20 transfer + ticket MINTED with payer, agentId, requestHash, interactionHash, endpoint. Revert ⇒ no ticket. Recover: ticketIdByTxHash.

4 — Handler x402

Only after mint succeeds. Agent server runs route, returns body. Computes responseHash.

5 — Finalize on-chain

finalizeTicket(ticketId, responseHash, agentReceiptSig?) — facilitator or agent (gas sponsored). Status FINALIZED. Agent may omit sig (censorship); client/facilitator can finalize without sig.

6 — Response x402

200 + resource + PAYMENT-RESPONSE (txHash, payer, ticketId). Optional X-X402-Interaction-Receipt header.

7 — Feedback on-chain

Client giveFeedbackWithTicket(...) only (legacy giveFeedback reverts). Requires FINALIZED ticket, msg.sender == payer. Dedup per (payer, agent): txHash + feedbackHash. Consumes ticket. Agent may disputeFeedback.
